# Expense Tracker System Report

This folder contains a GitHub Pages report site for the project. The site uses top-level Markdown pages so the Just the Docs sidebar stays clean. It is organized to match a typical university software engineering report structure and to work with the existing `.github/workflows/report-site.yml` workflow.

## Structure

- `_config.yml` configures the Jekyll site.
- `index.md` is the report home page.
- Top-level `*.md` files contain the main report chapters used by the site sidebar.
- `pictures/` is reserved for screenshots used in the final submission.
- `diagrams/` is reserved for architecture, database, and use-case diagrams.

## Before Final Submission

Update the following placeholders:

- Replace `[Student Name]` with your real name.
- Replace department, supervisor, and academic-year placeholders if needed.
- Add screenshots to `pictures/`.
- Add exported diagrams to `diagrams/`.
- Review the deployment URL in `_config.yml` if the GitHub repository name changes.
